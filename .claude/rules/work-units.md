# 작업 단위 규칙

## 기능 격리
- 커밋 1개 = 단일 관심사 (하나의 기능/수정)
- BE와 FE가 동시 변경되는 경우 하나의 커밋으로 묶어도 됨
- DB 마이그레이션은 별도 커밋

## 완료 체크리스트
- [ ] 코드 구현 완료
- [ ] 타입 체크 통과 (BE: mypy/pyright, FE: tsc --noEmit)
- [ ] 관련 문서 갱신 (docs/session/current-phase.md)
- [ ] 브라우저에서 동작 확인 (FE 변경 시)

## 에이전트 병렬 실행 조건
모든 조건 충족 시에만 병렬 가능:
- 수정 대상 파일 비중첩
- 모듈 독립 (BE/FE 분리)
- DB 마이그레이션 동시 진행 없음
- API 의존 관계 없음
